# CanchaLibre

## Objetivo

Este proyecto utiliza una guía de diseño estricta.

Antes de modificar cualquier interfaz debes consultar la documentación ubicada en `docs`.

## Orden de prioridad

1. UI_RULES.md
2. DESIGN_SYSTEM.md
3. COMPONENTS.md
4. LAYOUTS.md
5. BOOKING_FLOW.md
6. ANIMATIONS.md
7. ACCESSIBILITY.md
8. COPYWRITING.md
9. CODE_STYLE.md
10. FRONTEND_ARCHITECTURE.md

Nunca implementes componentes nuevos sin verificar primero si existe un componente reutilizable.

Toda nueva pantalla debe sentirse parte del mismo ecosistema visual.

Las interfaces deben parecer una plataforma premium similar a:

- Airbnb
- Playtomic
- Booking
- Stripe Dashboard

Nunca generar interfaces tipo panel administrativo.

Siempre reutilizar componentes.

Siempre respetar spacing, colores y tipografía definidos.

En caso de conflicto entre archivos prevalece:

UI_RULES.md.md
