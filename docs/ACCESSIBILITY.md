Cumplir WCAG AA.

Texto

4.5:1

Texto grande

3:1

Nunca utilizar grises con poco contraste.

44x44 px

Botones

52px

Inputs

52px

# Tipografía

Nunca menor a

14px

Texto principal

16px

# Focus

Todos los componentes deben tener

Focus visible

Outline

Primary Green

Nunca eliminar outline.

---

# Navegación

Toda la aplicación debe funcionar usando teclado.

Tab

Shift + Tab

Enter

Space

Escape

---

# Formularios

Todos los Inputs tienen

Label

Placeholder

Helper Text

Error

Nunca depender únicamente del placeholder.

# Iconos

Todo icono importante debe tener texto o aria-label.

---

# Imágenes

Todas las imágenes tienen alt.

Decorativas

alt=""

Informativas

Descripción clara.

Botón

aria-live

# Sidebar

Debe ser navegable mediante teclado.

# Calendario

Cada día debe poder seleccionarse usando teclado.

# Horarios

Cada horario debe ser un botón.

Nunca un div clickable.

aria-busy

200%

# Screen Readers

Todos los formularios deben tener

label

aria-describedby

cuando aplique.

# Motion

Si el usuario tiene activado

prefers-reduced-motion

Reducir todas las animaciones.

# Regla

La aplicación debe ser completamente utilizable sin mouse y comprensible para tecnologías de asistencia.

La accesibilidad no es opcional; forma parte del diseño desde el inicio.
